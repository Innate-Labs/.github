# Contributing · 贡献指南

Thanks for your interest in contributing to **Innate Labs** projects! 感谢你愿意参与共建 💛

## How to contribute · 如何贡献

1. **Open an issue first · 先开 issue** — For bugs or feature ideas, open an issue in the relevant repository so we can discuss before you invest time. 提 bug 或想法前，先在对应仓库开 issue 一起讨论。
2. **Fork & branch · Fork 并开分支** — `git checkout -b feat/your-feature`
3. **Keep PRs focused · PR 保持聚焦** — One logical change per pull request. 一个 PR 只做一件事。
4. **Describe your change · 写清说明** — Explain *what* and *why* in the PR description. 在 PR 描述里写清「改了什么、为什么」。

## Code style · 代码规范

- Match the style of the surrounding code. 跟随周围代码的既有风格。
- Run the project's lint/build before submitting. 提交前先跑通 lint / build。

## Code of Conduct · 行为准则

By participating you agree to abide by our [Code of Conduct](./CODE_OF_CONDUCT.md). 参与即表示你同意遵守我们的行为准则。
